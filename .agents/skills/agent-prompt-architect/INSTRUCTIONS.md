# Instructions

## When To Use

- criar um agent novo com utilidade real
- criar uma skill nova com gatilho e contrato claros
- melhorar o prompt de um agent antes de criá-lo
- revisar se um artifact proposto está genérico ou fraco
- estruturar melhor papel, responsabilidades e limites de um novo specialist

## Preferred Agent

`agent-prompt-architect-specialist`

## Workflow

1. entender o problema que motivou o novo artifact
2. decidir se o tipo correto é `agent`, `sub-agent` ou `skill`
3. fortalecer o prompt-base e o papel
4. definir responsabilidades, limites e contrato
5. só então preparar a criação no formato oficial

## Rules

- não criar artifact sem proposta de valor clara
- evitar nomes vagos e papéis amplos demais
- definir sempre limites operacionais e critérios de uso
- se a especificação ficar madura, usar o fluxo de factory para materializar o artifact
